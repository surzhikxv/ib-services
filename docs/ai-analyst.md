# ИИ-аналитик (Phase 1, п.4)

Раз в период модель готовит разбор простым языком по данным из озера, владелец может
задать вопрос своими словами. Разборы лежат в БД (`ai_reports`) и уходят в Telegram.

## Как устроено

```
данные озера → дайджест → промпт → модель → разбор/ответ → ai_reports → Telegram
   (вьюхи)     digest.py   prompts.py  llm.py    analyst.py     (БД)      telegram.py
```

- **digest.py** — компактный срез данных (KPI, воронка, выручка по тарифам/источникам,
  ряды по неделям). Модель видит цифры, а не сырые таблицы. Чистая выборка, под тестом.
- **prompts.py** — системный промпт (аналитик, простой язык, только по цифрам, без воды)
  и сборка промптов разбора/вопроса.
- **llm.py** — узкий интерфейс `LLMClient`. По умолчанию `AnthropicLLM` (Claude
  `claude-sonnet-5`, adaptive thinking). **Провайдер сменяем одной реализацией** — остальной
  код не меняется. `FakeLLM` — для тестов и dry-run.
- **analyst.py** — `generate_report` (еженедельный разбор) и `answer_question` (ответ на
  вопрос); пишут результат в `ai_reports` вместе с дайджестом.
- **telegram.py** — формат сообщения (под тестом) + отправка через Bot API (каркас).

## Запуск

```bash
# проба без ключа: показать дайджест и промпт, которые уйдут модели
python -m kontur.cli ai report --show-prompt

# боевой разбор (нужен LLM_API_KEY в .env)
python -m kontur.cli ai report --period 2026-W24
python -m kontur.cli ai ask "почему на этой неделе меньше продаж?"
```

`.env`: `LLM_API_KEY` (от клиента), `LLM_MODEL` (умолч. `claude-sonnet-5`),
`LLM_EFFORT` (умолч. `medium` — баланс цены и глубины для регулярного отчёта).

## Что нужно от клиента

- **Ключ языковой модели** (`LLM_API_KEY`) — без него считается всё, кроме самого вызова
  модели (`--show-prompt` работает уже сейчас).
- Токен Telegram-бота владельца (`TELEGRAM_BOT_TOKEN` + `TELEGRAM_CHAT_ID`) — для доставки.

## Расписание

Еженедельный запуск вешается в n8n: Cron → `app: python -m kontur.cli ai report` →
`telegram.send_telegram(...)`. В этой сессии — каркас доставки и сама генерация; узел n8n
настраивается при сборке оркестрации.

## Смена провайдера

Если ключ клиента — не Anthropic: добавить класс рядом с `AnthropicLLM` в `kontur/ai/llm.py`
с методом `complete(system, user) -> str` и собирать его в `cli._make_llm()`. Дайджест,
промпты, хранение и доставка не меняются.
