# Еженедельный ИИ-наставник

Каждый понедельник модель разбирает завершённую ISO-неделю и готовит план на следующие
7 дней: главное узкое место, до трёх контентных единиц, до двух экспериментов и одно действие
в воронке. Разбор сохраняется в `ai_reports`; при заданном chat ID отправляется в Telegram.

## Как устроено

```
данные озера → дайджест → промпт → модель → разбор/ответ → ai_reports → Telegram
   (вьюхи)     digest.py   prompts.py  llm.py    analyst.py     (БД)      telegram.py
```

- **digest.py** — компактный срез: свежесть площадок, baseline по площадке/формату/возрасту,
  свежие и сильные/слабые публикации, динамика каналов, KPI, воронка и качество атрибуции. `null` не
  подменяется нулём; Instagram явно помечается `unavailable`.
- **prompts.py** — системный промпт (аналитик, простой язык, только по цифрам, без воды)
  и сборка промптов разбора/вопроса.
- **llm.py** — узкий интерфейс `LLMClient`. По умолчанию `AnthropicLLM` (Claude
  `claude-sonnet-5`, adaptive thinking). **Провайдер сменяем одной реализацией** — остальной
  код не меняется. `FakeLLM` — для тестов и dry-run.
- **analyst.py** — `generate_report` (еженедельный разбор) и `answer_question` (ответ на
  вопрос); пишут результат в `ai_reports` вместе с дайджестом.
- **telegram.py** — формат сообщения (под тестом) + отправка через Bot API (каркас).

## Запуск

```bash
# проба без ключа: показать дайджест и промпт, которые уйдут модели
python -m kontur.cli ai report --show-prompt

# боевой разбор последней завершённой недели
python -m kontur.cli ai report --previous-week --send
# ручной повтор за тот же период
python -m kontur.cli ai report --previous-week --force
python -m kontur.cli ai ask "почему на этой неделе меньше продаж?"
```

`.env`: `LLM_API_KEY` (от клиента), `LLM_MODEL` (умолч. `claude-sonnet-5`),
`LLM_EFFORT` (умолч. `medium` — баланс цены и глубины для регулярного отчёта).

## Что нужно от клиента

- **Ключ языковой модели** (`LLM_API_KEY`) — без него считается всё, кроме самого вызова
  модели (`--show-prompt` работает уже сейчас).
- Токен и chat ID получателя (`AI_TELEGRAM_BOT_TOKEN` + `AI_TELEGRAM_CHAT_ID`;
  если пусто — fallback на `TELEGRAM_*`).

## Расписание

`kontur-ai-mentor.timer` проверяет задачу ежедневно в `04:00 UTC` / `07:00 МСК`, после утренней синхронизации. В понедельник
создаётся отчёт за прошлую ISO-неделю; в остальные дни таймер лишь повторяет незавершённую генерацию или доставку.
Повторная генерация за тот же `period` пропускается, поэтому дополнительных вызовов модели нет. Метка Telegram-доставки пишется
только после успеха. Для осознанной повторной генерации есть `--force`.

## Смена провайдера

Если ключ клиента — не Anthropic: добавить класс рядом с `AnthropicLLM` в `kontur/ai/llm.py`
с методом `complete(system, user) -> str` и собирать его в `cli._make_llm()`. Дайджест,
промпты, хранение и доставка не меняются.
