# Дашборд Metabase — настройка

Логика дашборда лежит в БД как SQL-вьюхи (`kontur/dashboard/views.py`), а карточки —
в каталоге (`kontur/dashboard/catalog.py`). Поэтому собрать панель можно двумя путями:
автоматически (скрипт) или руками (по таблице ниже). Боевая БД — Postgres из
`docker-compose`; вьюхи создаются сами при `db init` и старте API.

## Путь 1. Автоматически (на VPS, когда поднят compose)

```bash
docker build --build-arg VCS_REF=local -t kontur-app:latest .
docker compose up -d
# 1) один раз пройти мастер Metabase на http://<сервер>:3000 — создать админа
# 2) вписать в .env: METABASE_USER / METABASE_PASSWORD (URL уже http://metabase:3000)
docker compose run --rm app python -m kontur.cli metabase provision
```

Скрипт залогинится, добавит наш Postgres как источник, создаст вопросы из каталога
и соберёт дашборд «Контур роста — обзор» в отдельной коллекции «Контур роста».
Повторный запуск обновляет существующие карточки и перемещает их в эту коллекцию
(идемпотентно по имени). Ориентир по API — Metabase v0.48+.

Демонстрационные `Sample Database`, коллекция `Examples` и дашборд
`E-commerce Insights` к проекту не относятся. После первоначальной настройки их
можно отправить в архив через интерфейс Metabase; провижининг их не использует.

> Автонастройка — ускоритель. Если версия Metabase разойдётся с API, надёжный путь —
> ручная сборка ниже: SQL карточек тот же, он уже проверен на данных.

## Путь 2. Руками (10–15 минут)

1. **Источник данных:** Admin → Databases → Add → PostgreSQL. Host `postgres`, port `5432`,
   база/логин/пароль — из `.env`. Сохранить, дождаться синхронизации схемы.
2. **Карточки:** New → SQL query → выбрать нашу БД → вставить SQL из колонки ниже →
   Visualization выбрать по колонке «Тип» → Save.
3. **Дашборд:** New → Dashboard «Контур роста — обзор» → добавить карточки.
   Раскладка: 5 KPI-скаляров верхней строкой, ниже в две колонки — воронка и деньги,
   ещё ниже — динамика по неделям.

### Карточки дашборда

| Карточка | Тип | Вьюха | Назначение |
|---|---|---|---|
| Подписчиков | `scalar` | `v_kpis` | Всего людей в боте |
| Платящих | `scalar` | `v_kpis` | Уникальных покупателей |
| Конверсия в оплату, % | `scalar` | `v_kpis` | Платящие / подписчики |
| Оплат | `scalar` | `v_kpis` | Всего оплат (тариф можно купить не один) |
| Выручка, ₽ | `scalar` | `v_kpis` | Заработает при заданных ценах тарифов или суммах из вебхука Prodamus |
| Воронка по этапам | `funnel` | `v_funnel` | Уникальные пользователи, дошедшие до каждого этапа |
| Оплаты и выручка по тарифу | `bar` | `v_revenue_by_tariff` | Базовый / Стандарт / Премиум |
| По источникам трафика | `row` | `v_revenue_by_source` | UTM start-ссылки по каналам; входы без метки — `direct` |
| Новые подписчики по неделям | `line` | `v_subscribers` | В Metabase можно переключать гранулярность (день/неделя/месяц) |
| Оплаты и выручка по неделям | `line` | `v_payments` | Точное время оплаты добирается вебхуком Prodamus |
| Свежесть источников | `table` | `v_connector_freshness` | Последний успешный или ошибочный запуск каждого коннектора |

SQL каждой карточки — в `kontur/dashboard/catalog.py` (поле `metabase_sql`).
Для большинства карточек это `SELECT ... FROM <вьюха>`, его же можно вставить руками.

## Откуда берётся выручка (₽)

Выручка считается как `COALESCE(оплата.сумма, тариф.цена, 0)`. Сначала используется
реальная сумма из вебхука Prodamus, а если её нет — цена тарифа. Чтобы цифры были
ненулевыми, достаточно одного из:
- проставить цены тарифов: `UPDATE tariffs SET price = <₽> WHERE key IN ('basic','standard','premium');`
- или включить приём сумм из вебхука Prodamus (каркас приёмника готов: `POST /webhooks/prodamus`).

## Как проверить живой дашборд

Не фиксируйте текущие счётчики в документации: они меняются после каждого входа в
бот и оплаты. Проверка после провижининга:

1. Открыть «Контур роста — обзор» и убедиться, что все 11 карточек загрузились без ошибок.
2. Сверить верхние KPI с `SELECT * FROM v_kpis`.
3. Сверить этапы с `SELECT * FROM v_funnel ORDER BY ordering`.
4. Проверить, что в «Свежести источников» присутствуют Telegram, TikTok, VK и YouTube.
5. Проверить, что `direct` отображается как «(прямой вход)», а не отдельный технический код.
