# VK — как достать доступы (пошагово)

Коннектору VK нужны **ДВА токена** + numeric id сообщества. Почему два: community-токен
отдаёт **контент** (посты, лайки, подписчиков), но **НЕ отдаёт статистику** (охваты/просмотры/
визиты) — для неё нужен отдельный **user-токен админа** со scope `stats`. Это ключевой нюанс:
на одном community-токене охватов не будет.

Итог — три значения в `.env`:
```
VK_GROUP_ID=            # числовой id сообщества (без минуса)
VK_COMMUNITY_TOKEN=     # контент: посты, подписчики
VK_USER_STATS_TOKEN=    # статистика: охваты/просмотры
```

> ⚠️ **Домены VK переезжают** с `vk.com` на `vk.ru` (плановый cutover ~30 сентября 2026).
> Сейчас работает `vk.com`; если что-то редиректит/не открывается — пробуйте тот же путь на `vk.ru`.
> Минтить токен и звать API нужно на **одном** семействе доменов.

---

## Шаг 0. Узнать numeric id сообщества → `VK_GROUP_ID`

Откройте сообщество. Если в адресе `vk.com/club123456789` — число и есть id. Если короткое имя
(`vk.com/samodvizhenie`) — откройте «…» → «Скопировать ссылку», либо https://regvk.com/id/ ,
либо позже возьмём через `groups.getById`. Сохраните **число без минуса** в `VK_GROUP_ID`.
(В запросах к стене коннектор сам подставит `owner_id = -VK_GROUP_ID`.)

---

## Шаг A. Community-токен (контент) → `VK_COMMUNITY_TOKEN`

Вы должны быть **админом/создателем** сообщества.

1. Откройте сообщество → правое меню **«Управление»**.
2. **«Настройки»** → вкладка **«Работа с API»** → вкладка **«Ключи доступа»** → **«Создать ключ»**.
   (Прямая ссылка: `https://vk.com/clubXXXXXX?act=tokens`, где `XXXXXX` — числовой id.)
3. В диалоге отметьте права: **«Доступ к стене»** (wall) и **«Управление сообществом»** (manage).
   Поля «Статистика» здесь нет — это нормально (см. Шаг B).
4. Подтвердите по SMS/2FA → **скопируйте токен** (длинная строка, начинается с `vk1.a...`).
   Положите в `VK_COMMUNITY_TOKEN`.

Community-токен по умолчанию **бессрочный**.

---

## Шаг B. User-токен админа со статистикой → `VK_USER_STATS_TOKEN`

Это токен **вашего личного аккаунта** (как админа сообщества) со scope `stats,offline`.
Старое быстрое приложение «VK Admin» отключили **13.04.2026**, поэтому регистрируем своё.

1. Зайдите на **https://dev.vk.com/** под аккаунтом-админом сообщества → **«Мои приложения»** →
   **«Создать»** → тип **«Standalone-приложение (SDK)»** → задайте любое имя → создать.
   Запомните **ID приложения** (`client_id`).
2. В настройках приложения задайте **«Базовый домен»** и добавьте доверенный redirect URI —
   например `https://oauth.vk.com/blank.html`.
3. Откройте в браузере (залогинены тем же аккаунтом-админом) такую ссылку, подставив свой `client_id`:
   ```
   https://oauth.vk.com/authorize?client_id=ВАШ_CLIENT_ID&display=page&redirect_uri=https://oauth.vk.com/blank.html&scope=stats,offline&response_type=token&v=5.199
   ```
   `scope=stats` даёт статистику по сообществам, где вы админ; `offline` делает токен **бессрочным**.
4. Нажмите «Разрешить». VK перебросит на пустую страницу, а **токен будет в адресной строке** после
   `#access_token=...` (до `&expires_in`). Скопируйте это значение в `VK_USER_STATS_TOKEN`.

> Запасной путь, если возиться с приложением неохота: минт-помощник `https://vkhost.github.io/`
> → «Настройки» → права `stats` + `offline`. Но своё Standalone-приложение — поддерживаемый путь.

> ⚠️ **«application is blocked» при авторизации?** Значит выбранное приложение заблокировано VK.
> `client_id=6121396` — это **VK Admin** (отключён 13.04.2026), его vkhost подставляет по умолчанию.
> Фикс: используйте **Kate Mobile** (`client_id=2685278`) тем же запросом:
> `https://oauth.vk.com/authorize?client_id=2685278&scope=stats,offline&redirect_uri=https://oauth.vk.com/blank.html&display=page&response_type=token&v=5.199`
> Если и он заблокирован — заведите **своё** Standalone-приложение (Шаг B выше) и подставьте его
> `client_id`: своё приложение VK не блокирует.

## Шаг C. IP-allowlist (возможно, не понадобится)

В 2026 VK иногда требует привязать токен к IP вызывающего сервера. **Сначала просто пробуем без
этого.** Если коннектор начнёт получать ошибку доступа (code 15) — добавьте в настройках ключа
IP нашего сервера **147.45.245.137** в список разрешённых. (VPS временный — при переезде IP
поменяется, это в runbook миграции.)

---

## Безопасность

- Оба токена — **секреты**: только в `.env` (он в `.gitignore`), в код/гит не попадают.
- Если токен где-то засветился — перевыпустите (community-токен — заново в «Работа с API»,
  user-токен — повторив Шаг B; благодаря `offline` старый можно отозвать).
