# VK — как достать доступы (пошагово)

Коннектору VK нужен **ОДИН токен** + numeric id сообщества. Так вышло по итогам
живой проверки API:

- **community-токен сообщества НЕ умеет читать стену** — `wall.get` на нём падает с
  «method is unavailable with group auth». То есть собрать посты им нельзя.
- **user-токен админа** (ваш личный аккаунт, где вы админ сообщества) умеет ВСЁ, что
  нам нужно: посты+реакции (`wall.get`), охваты постов (`stats.getPostReach`),
  дневную статистику сообщества (`stats.get`), карточку сообщества (`groups.getById`).

Поэтому два значения в `.env`:
```
VK_GROUP_ID=            # числовой id сообщества (без минуса)
VK_USER_STATS_TOKEN=    # user-токен админа: посты + охваты + статистика
```
(`VK_COMMUNITY_TOKEN` больше не нужен — можно удалить.)

> ⚠️ **Домены VK переезжают** с `vk.com` на `vk.ru` (плановый cutover ~30 сентября 2026).
> Сейчас работает `vk.com`; если что-то редиректит/не открывается — пробуйте тот же путь
> на `vk.ru`. Минтить токен и звать API нужно на **одном** семействе доменов.

---

## Шаг 0. Узнать numeric id сообщества → `VK_GROUP_ID`

Откройте сообщество. Если в адресе `vk.com/club123456789` — число и есть id. Если короткое
имя (`vk.com/s.lapychev`) — откройте «…» → «Скопировать ссылку», либо https://regvk.com/id/ .
Сохраните **число без минуса** в `VK_GROUP_ID`. (В запросах к стене коннектор сам
подставит `owner_id = -VK_GROUP_ID`.)

---

## Шаг 1. User-токен админа → `VK_USER_STATS_TOKEN`

Это токен **вашего личного аккаунта** (как админа сообщества). Старое быстрое приложение
«VK Admin» отключили **13.04.2026**, поэтому регистрируем своё Standalone-приложение.

1. Зайдите на **https://dev.vk.com/** под аккаунтом-админом сообщества → **«Мои приложения»**
   → **«Создать»** → тип **«Standalone-приложение (SDK)»** → задайте любое имя → создать.
   Запомните **ID приложения** (`client_id`).
2. В настройках приложения добавьте доверенный redirect URI — `https://oauth.vk.com/blank.html`.
3. Откройте в браузере (залогинены тем же аккаунтом-админом) ссылку, подставив свой `client_id`:
   ```
   https://oauth.vk.com/authorize?client_id=ВАШ_CLIENT_ID&display=page&redirect_uri=https://oauth.vk.com/blank.html&scope=stats,offline&response_type=token&v=5.199
   ```
   `scope=stats` даёт статистику по сообществам, где вы админ; `offline` делает токен
   **бессрочным**. Чтение стены публичного сообщества (`is_closed=0`) работает и без отдельного
   wall-scope — стена публична. Если сообщество закрытое, добавьте в scope `wall,groups`.
4. Нажмите «Разрешить». VK перебросит на пустую страницу, а **токен будет в адресной строке**
   после `#access_token=...` (до `&expires_in`). Скопируйте это значение в `VK_USER_STATS_TOKEN`.

> Запасной путь, если возиться с приложением неохота: минт-помощник `https://vkhost.github.io/`
> → права `stats` + `offline`. Но своё Standalone-приложение — поддерживаемый путь.

> ⚠️ **«application is blocked» при авторизации?** Значит выбранное приложение заблокировано VK.
> `client_id=6121396` — это **VK Admin** (отключён 13.04.2026), его vkhost подставляет по
> умолчанию. Фикс: используйте **Kate Mobile** (`client_id=2685278`) тем же запросом, либо
> заведите **своё** Standalone-приложение (Шаг 1) — своё VK не блокирует.

## Шаг 2. IP-allowlist (возможно, не понадобится)

В 2026 VK иногда требует привязать токен к IP вызывающего сервера. **Сначала просто пробуем
без этого.** Если коннектор начнёт получать ошибку доступа (code 15) — добавьте в настройках
ключа IP нашего сервера **147.45.245.137** в список разрешённых. (VPS временный — при переезде
IP поменяется, это в runbook миграции.)

## Проверка, что токен живой

С любой машины (VK — РФ-родной, релей не нужен):
```bash
curl -s "https://api.vk.com/method/groups.getById?group_id=ВАШ_GROUP_ID&fields=members_count&access_token=ВАШ_ТОКЕН&v=5.199"
```
Ждём `{"response":{"groups":[{...,"members_count":N}]}}`. `{"error":...}` в теле (HTTP 200) —
токен/права не те.

## Безопасность

- Токен — **секрет**: только в `.env` (он в `.gitignore`), в код/гит не попадает.
- Если токен где-то засветился — перевыпустите (повторите Шаг 1; благодаря `offline`
  старый сеанс можно отозвать в «Настройки аккаунта → Безопасность → История активности»).
- Токен из Kate Mobile широкий (даёт и доступ к личным сообщениям) — для чистоты можно
  перевыпустить узкий через **своё** Standalone-приложение со `scope=stats,offline`.

## Что прислать мне

Достаточно двух значений: `VK_GROUP_ID` и `VK_USER_STATS_TOKEN`. Пришлите — впишу в `.env`
(не в гит) и запущу `kontur vk sync`. Если на каком-то шаге застрянете (особенно редиректы
доменов / «application is blocked») — напишите, на чём именно, подскажу.
