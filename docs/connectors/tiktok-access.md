# TikTok — как достать данные в озеро (B+)

TikTok органику не отдаёт: внутренние запросы Studio подписаны (`webmssdk`/X-Bogus),
серверно их не воспроизвести, а Business API владельцем отклонён. Поэтому коннектор
`kontur.tiktok` — **ингест файлов**, снятых из залогиненного браузера владельца.
Режим **B+**: userscript в его реальном браузере сам обходит видео и заливает JSON
на сервер. Без headless, без передачи сессии, человеческим темпом — выглядит как он
сам листает Studio.

| Источник | Что даёт | Куда в озере |
|---|---|---|
| **capture-JSON** (userscript) | per-video: views, **reach**, likes, comments, **saves**, watch-time, % досмотра, новые подписчики, **источники трафика**, retention-кривая, поисковые запросы, **демография** (пол/возраст/гео) | `content` + `content_metrics` (6 колонок + `raw`) |
| **Overview.csv** (нативный экспорт) | канал-дневная: video/profile views, likes, comments, shares | `channel_metrics` |

## Эндпоинт (сервер, один раз)
1. Сгенерировать токен: `python -c "import secrets; print(secrets.token_urlsafe(32))"`.
2. Положить в `.env`: `TIKTOK_INGEST_TOKEN=…`, перезапустить API-сервис.
3. Эндпоинты: `POST /ingest/tiktok` (заголовок `X-Kontur-Token`), `GET /ingest/tiktok/health`
   (возраст последнего sync; `stale=true`, если данных нет >8 дней — основа алерта).

## A. Авто-обход в браузере (capture-JSON) — основной поток

### 0. Установка (Chrome, один раз) — ВАЖНО про MV3
Современный Chrome (Manifest V3) НЕ запускает userscript'ы, пока не включены **две**
галки — без них Tampermonkey покажет скрипт «включённым», но он не выполнится
(плашки не будет, ошибок в консоли тоже):
1. Tampermonkey → Dashboard → **«+»** → вставить `tiktok-capture.user.js` → Ctrl/Cmd-S.
2. `chrome://extensions` → вверху справа **«Режим разработчика» → ВКЛ**.
3. Там же: карточка Tampermonkey → **«Подробнее»** → **«Разрешить пользовательские скрипты» → ВКЛ**.
   (В Firefox/Violentmonkey этих шагов нет — достаточно установить скрипт.)

### 1. Сбор
1. Открыть **desktop** `tiktok.com/tiktokstudio` (войти), дождаться плашки «Контур · TikTok B+» снизу справа.
2. В плашке вписать **Endpoint** (напр. `https://thedialog.ru/ingest/tiktok`) и **Token** (= `TIKTOK_INGEST_TOKEN`).
3. Открыть **«Публикации»** и **один раз медленно промотать список до самого конца**.
   Список в DOM виртуализирован (всегда ~8 строк, синтетической прокруткой не
   вычерпывается), поэтому перечисление берётся из ответов `item_list`, которые
   страница подгружает **только при настоящей прокрутке**. Промотал — хук поймал все
   страницы каталога (это сразу и базовые счётчики по каждому видео: показы/лайки/
   комменты/репосты/сохранения).
4. **«Сухой прогон»** → плашка покажет число найденных видео. Меньше, чем в «Публикациях»?
   Домотай список ниже и нажми ещё раз. Сошлось — дальше.
5. **«Старт обход»** → скрипт по очереди открывает каждое видео и его 3 вкладки
   (Обзор/Зрители/Вовлечённость) с паузами 5–9 c, копит **богатые** данные (охват,
   watch-time, % досмотра, retention, источники трафика, демография). По концу —
   **сам зальёт** JSON на Endpoint (или «Залить»/«Скачать» вручную). После успешной
   заливки буфер очищается.

> Темп щадящий, всё в его сессии. Видео, которые обход не прошёл, всё равно попадут
> в озеро с базовыми счётчиками (из `item_list`). Сначала всегда «Сухой прогон». На паузу — «Стоп».

## B. Overview.csv (канал-дневная)
1. `tiktok.com/tiktokstudio` → Analytics → **Overview** → диапазон **Last 60 days** → **Export** (zip).
2. Имя zip содержит дату начала (`Overview_2026-04-28_…`) — это `--year`. Экспортировать
   **раз в ≤30 дней** (окно 60, backfill за его пределами невозможен).
3. Залить вручную: `kontur tiktok sync --overview Overview.csv --year 2026`.
   (Заливку Overview через расширение добавим позже; пока — CLI.)

## Ручной режим / отладка
- Без авто-обхода: на странице видео прокликать 3 вкладки руками → «Скачать» → файл →
  `kontur tiktok sync --capture tiktok_capture_YYYY-MM-DD.json`.
- Идемпотентно: повтор того же дня перезаписывает снимки (одна строка на сущность/UTC-день).

## Каденция и надёжность
- **capture** — еженедельно (авто-обход в его браузере).
- **Overview** — раз в ≤30 дней.
- Застой ловим через `GET /ingest/tiktok/health` (`stale=true` → попросить перелогиниться/прогнать).
- Аккаунт: `@lapychevdcp` (user_id `7362975467459380230`). URL аналитики видео:
  `…/analytics/{aweme_id}` (обзор) · `/{aweme_id}/viewers` · `/{aweme_id}/engagement`.

## Риски (костыль, честно)
- ToS-серость (автоматизация залогиненного аккаунта) — но пассивное чтение + человеческий
  темп + его реальный браузер/IP → бан-риск низкий. Не headless, сессию не отдаём.
- Хрупкость: TikTok меняет внутренние endpoint'ы/вёрстку без анонса → скрипт чиним.
  Перечисление идёт из ответов `item_list` (надёжнее DOM), но требует ручной
  прокрутки «Публикаций» до конца — синтетический скролл react-window игнорирует.
  «Сухой прогон» показывает, всё ли поймано.
- Подпись запросов (`byted_acrawler.frontierSign`) на странице есть, но мы её НЕ
  трогаем: только пассивно читаем ответы. Форж своих signed-запросов = риск бана
  основного аккаунта — сознательно не делаем.
