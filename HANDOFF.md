# HANDOFF — Контур роста проекта (старт сборки Phase 1)

Этот файл — самодостаточный бриф для НОВОЙ сессии. Прочитай его целиком, дальше можно строить без предыдущей переписки.

## Что за проект
MVP системы аналитики и ИИ-курирования для инфобизнеса.
- **Клиент:** Сергей Лапычев — курсы реабилитации детей с ДЦП. Трафик органический: TikTok (главный), YouTube, Instagram, VK; продажи и воронка — в Telegram через бота.
- **Подрядчик:** seva (thedialog.kz).
- **Суть системы:** «озеро данных + ИИ над ним». Собрать данные всех каналов в одну базу → ИИ даёт разборы и рекомендации → плюс помощь с контентом, коллабами, анализом ниши.
- **Бюджет:** 50–70k ₽ разово за MVP + хостинг и ИИ-API ежемесячно отдельно (платит клиент). Срок MVP — 3–4 недели.

## Задача ЭТОЙ сессии (только пункты 1–2)
1. **Фундамент.** Репозиторий, `docker-compose` (PostgreSQL + n8n + Metabase + приложение), схема **озера данных** (таблицы: каналы, контент, события, подписчики, шаги воронки, оплаты, источники), единая событийная модель воронки.
2. **Коннектор BotHelp на живых данных.** Авторизация с авто-обновлением токена; выгрузка ботов, подписчиков (пагинация), шагов воронки; маппинг 28 шагов в этапы воронки; распознавание тарифов и оплат; запись в базу. Скрипт/CLI для запуска выгрузки.

Не делать в этой сессии: остальные коннекторы (только каркасы при желании), дашборд-панели, продакшн-деплой.

## Технологический стек (решено)
Python / FastAPI · PostgreSQL · n8n (оркестрация) · Metabase (дашборд) · LLM по API · Telegram-бот владельца (aiogram) · хостинг VPS (позже).

## Зафиксированные решения
- Бот и воронка клиента — в конструкторе **BotHelp**. Мы не пересобираем бота, а **надстраиваемся** над ним.
- Оплаты — **Prodamus, интегрирован внутрь BotHelp**. Отдельный эквайринг не подключаем; события оплат берём из BotHelp (и/или прямой вебхук Prodamus как сверка).
- Поэтапной аналитики BotHelp по API не отдаёт — **аналитику воронки строим у себя** из подписчиков/тегов/событий. Живые события (оплата, тег, проход шага) лучше принимать **вебхуком**, а не опросом API.
- **Instagram:** автосбор только если аккаунт Business/Creator (Graph API). Тип аккаунта у клиента ещё не подтверждён. Если личный — ручные выгрузки.
- **TikTok:** органическую аналитику официально не отдают → ручные выгрузки (полуавтомат) или платный парсер. По умолчанию — ручные выгрузки.
- **Контент-помощник (нарезка/монтаж видео) — Phase 2**, в этот MVP не входит.

## BotHelp — проверено вживую (2026-06-15)
Open API на тарифе клиента РАБОТАЕТ (проверено реальными запросами). Креды лежат в `.env` (см. ниже).

**Авторизация:**
- POST `https://oauth.bothelp.io/oauth2/token`, form-data: `grant_type=client_credentials`, `client_id`, `client_secret`.
- Ответ: `access_token` (Bearer), `expires_in: 3600`. Токен живёт 1 час, обновляется тем же запросом.
- Заголовок к API: `Authorization: Bearer <token>`. Базовый URL: `https://api.bothelp.io`.

**GET-эндпоинты (внимание на слэши!):**
- `GET /v1/bots/` — со слэшем. Вернул: один бот `{title: "Курс", referral: "c1778263208250"}`.
- `GET /v1/funnels/` — со слэшем. Пусто `[]` (воронка не как «funnel», а как шаги бота).
- `GET /v1/subscribers/` — со слэшем. Курсорная пагинация: `paging.cursor.after`, `paging.next` (напр. `?after=34`). Данные подписчиков отдаются.
- `GET /v1/bots/{bot_referral}/steps` — БЕЗ слэша (со слэшем 301 редиректит). Вернул 28 шагов.

**Структура воронки (28 шагов бота «Курс»):** Приветствие → Выбор пакета → Инфо о пакетах 1/2/3 → три тарифа **премиум / базовый / стандарт** с шагами «Оплата премиум/базовый/стандарт» (тут ссылки Prodamus) и «удаление премиум/база/стандарт» (снятие доступа/тегов) + служебные «Действия», «Задержка», «Сообщение».

**Swagger:** `https://main.bothelp.io/swagger/api.json` (18 путей; 4 GET выше; остальное POST/PUT/DELETE — управление подписчиками, теги, потоки/последовательности, рассылки).

## Безопасность
Секрет BotHelp засветился при проверке. Креды — только в локальном `.env` (в `.gitignore`). Когда нужно — клиент перевыпустит их в BotHelp → Настройки → Интеграции → Open API → «ОБНОВИТЬ».
